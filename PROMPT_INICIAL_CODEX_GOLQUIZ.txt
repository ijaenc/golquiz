Quiero que trabajes sobre este proyecto Flutter llamado GolQuiz.

Antes de modificar código:

1. Lee completo el archivo `CODEX_CONTEXT_GOLQUIZ.md` ubicado en la raíz del proyecto.
2. Revisa el árbol actual del repositorio.
3. Revisa `pubspec.yaml`.
4. Revisa todo el contenido existente dentro de `lib/`.
5. No asumas que los archivos están completos solo porque existen.

Contexto principal:
- Flutter + Dart.
- Estado con Provider y MultiProvider.
- Primera versión local con usuario demo.
- SharedPreferences para persistencia.
- Supabase se integrará después.
- El diseño debe seguir el mockup de Figma indicado en el documento.
- No usar Riverpod, Bloc ni Tailwind.
- No implementar todavía Supabase, API externa ni grupos.

Objetivo de esta iteración:
Dejar funcionando la base completa del MVP local de GolQuiz.

Implementa en orden:

1. Tema global y constantes visuales.
2. Modelos necesarios.
3. LocalStorageService.
4. QuestionService con preguntas locales.
5. AuthProvider.
6. ProfileProvider.
7. QuizProvider.
8. RankingProvider.
9. MultiProvider en `main.dart`.
10. Pantalla Welcome.
11. Home con navegación inferior.
12. Categorías.
13. Modal inferior para elegir:
   - dificultad fácil, media o difícil;
   - 5 o 10 preguntas.
14. Quiz funcional.
15. Retroalimentación correcta o incorrecta.
16. Resultado final.
17. Ranking local con usuarios mock y usuario demo dinámico.
18. Perfil editable.
19. Persistencia local.
20. Navegación completa.

Reglas importantes:
- Mantén la arquitectura existente.
- Usa widgets reutilizables.
- Evita lógica extensa dentro de las pantallas.
- No agregues dependencias salvo que sean realmente necesarias.
- No inicialices Supabase.
- No borres placeholders útiles para integración futura.
- Las preguntas deben mezclarse sin repetirse dentro de la partida.
- No debe existir temporizador.
- Una respuesta incorrecta no resta puntos.
- Puntaje:
  - fácil: 10;
  - media: 20;
  - difícil: 30.
- Bono de racha:
  - 2 correctas: +5;
  - 3 correctas: +10;
  - 4 o más: +15.
- Debe guardarse puntaje total y mejor resultado por categoría, dificultad y cantidad.
- El nombre editado debe reflejarse en Home, Ranking y Perfil.

Al terminar:

1. Ejecuta `flutter pub get`.
2. Ejecuta `flutter analyze`.
3. Corrige todos los errores.
4. Si existen tests, ejecútalos.
5. Resume:
   - archivos creados;
   - archivos modificados;
   - funcionalidades terminadas;
   - pendientes;
   - posibles riesgos.
6. Sugiere un mensaje de commit.

No hagas un cambio masivo sin revisar primero el código actual.
Avanza de forma incremental y deja la aplicación ejecutable al final.
